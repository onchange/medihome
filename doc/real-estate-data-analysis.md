## Phase 1での利用可能性評価

### ✅ 利用可能なデータ

1. **地区別相場表示**
   - `DistrictName` × `TradePrice` で地区別平均価格を算出可能
   - 例: 「日の出エリアの平均取引価格: 5,800万円」

2. **医療アクセススコアとの統合**
   - `DistrictName` または `DistrictCode` で医療施設データと紐付け可能
   - 例: 「日の出エリア：医療スコア75点、平均取引価格5,800万円」

3. **価格帯フィルタリング**
   - `TradePrice` で価格帯別に物件を分類可能
   - 例: 「4,000万円〜6,000万円の物件を検索」

4. **物件タイプ別分析**
   - `Type` でマンション/戸建てを区別可能

5. **間取り別分析**
   - `FloorPlan` で３ＬＤＫ、４ＬＤＫなどに分類可能

### ⚠️ Phase 1では未実装（Phase 2以降）

- **緯度経度情報なし**: 地図上にピン表示できない
  - 回避策: 地区名（`DistrictName`）で集計し、エリア単位で表示
  - Phase 2: ジオコーディングAPIで緯度経度を取得

- **詳細住所なし**: 「日の出1丁目」などの細かい住所は取得できない
  - 回避策: 地区レベル（「日の出」「高洲」など）で集計

## Phase 1実装方針

### 推奨アプローチ

**Phase 1では地区レベルの統合に注力**

1. **地区別集計**
   ```typescript
   interface DistrictStats {
     districtName: string      // 地区名（例：日の出）
     districtCode: string      // 地区コード
     averagePrice: number      // 平均取引価格
     medianPrice: number       // 中央値
     transactionCount: number  // 取引件数
     medicalScore: number      // 医療アクセススコア（医療データから算出）
   }
   ```

2. **医療施設データとの紐付け**
   - 医療施設の住所から地区名を抽出（例: 「千葉県浦安市日の出1-2-3」→「日の出」）
   - 地区ごとに医療施設数をカウント
   - 医療アクセススコアを算出

3. **UI表示例**
   ```
   【日の出エリア】
   医療アクセススコア: 75点
   - 小児科: 3件（半径1km）
   - 総合病院: 1件（徒歩15分）

   不動産相場:
   - 平均取引価格: 5,800万円
   - 中央値: 5,400万円
   - 取引件数: 75件（2024年）
   ```

### Phase 2での拡張

- ジオコーディングAPIで緯度経度を付与
- 地図上に個別物件をピン表示
- 物件単位での医療アクセススコア算出（距離計算）
